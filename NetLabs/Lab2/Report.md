# Лабораторная работа №2
**Сравнение сервисов Amazon Web Services и Microsoft Azure. Создание единой кросс-провайдерной сервисной модели.**  
**Вариант 7**  
**Выполнил:** Шитарев Андрей

## Задача 
На основе данных биллинга Microsoft Azure нужно было заполнить первые пять колонок таблицы точно так же, как это было сделано в предыдущей лабе с AWS. Главное было сохранить единую модель классификации, использовать те же самые названия, чтобы потом можно было сравнивать сервисы двух провайдеров.

Для ориентира у меня уже был готовый заполненный файл из первой лабы (соседняя ветка репозитоия), где всё было разбито по AWS-сервисам.

## Ход работы
Работал тоже в Microsoft Excel.
Для лосьтижения цели сделал следующее:

1. Открыл csv файл
2. Рядом открыл заполненный AWS-файл, чтобы смотреть на предыдущие названия
3. Для каждой строки смотрел, какой сервис Azure используется (по колонкам)
4. Подбирал подходящие значения для колонок A–E:
   - Для **IT Tower** брал те же категории, что были в AWS лабе
   - Для **Service Family** тоже старался использовать похожие 
   - Для **Service Type** писал название конкретного сервиса Azure 
   - Для **Service Sub Type** и **Service Usage Type** описывал точнее, что именно потребляется 

Все строки обработал вручную. Ориаентируясб на то, чтобы названия IT Tower и Service Family совпадали с теми, что были в AWS-файле.

## Выводы
В процессе работы я увидел, насколько похожи сервисы Azure и AWS, и как их можно привести к одной общей модели классификации. Когда оба файла заполнены по одной схеме, можно легко сравнивать расход ресурсов между провайдерами.
