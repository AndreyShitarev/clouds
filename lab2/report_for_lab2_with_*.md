## Разберёмся почему это работает 

__Что вообще значит не видят друг друга по сети?__
```
Когда контейнеры работают в одной сети, Docker предоставляет:
- виртуальный сетевой интерфейс для каждого контейнера
- мостовую сеть для связи контейнеров
- встроенный DNS, который разрешает имена сервисов в IP-адреса контейнеров
- маршрутизацию между контейнерами в этой сети
```

__Если два контейнера не подключены к одной сети, встроенная маршрутизация и DNS не позволят им "общаться друг с другом". Так и работает изоляция. Контейнеры запущены в одном проекте и могут быть подняты одной командой, но сетевые пакеты не могут проходить между ними.__

## Как я этого добился
__Я явно определил два пользовательских стевых драйвера__
```
networks:
  web_net:
    driver: bridge
  db_net:
    driver: bridge

```

__Для каждого сервиса указал только одну сеть__
```
services:
  web:
    networks:
      - web_net

  db:
    networks:
      - db_net
```

__Багодаря этому__
```
- web контейнер получает интерфейс в web_net и в её подсети IP 
- db контейнер получает интерфейс в db_net и в её подсети IP 
- нет общей сети, поэтому Docker не добавляет маршрутов между ними и DNS и не разрешает имена одного сервиса внутри другого
```

## Итог
- Сети Docker - это отдельные network namespaces. Контейнер в одной сети имеет свой виртуальный интерфейс и таблицу маршрутов. Другой контейнер в другой сети имеет другой отдельный интерфейс и другая таблица маршрутов. Без маршрута пакеты данных не идут между ними.
- Встроенный DNS привязан к каждой сети. Он знает контейнеры, подключённые к каждой сети, и разрешает их имена только в этой же сети.
